Architecture & design

Two ideas carry the whole school: layering (the baseline stays small; opinions live downstream) and a single-gate docs taxonomy (the design record this site is built from).

The charter — and its edges

ACE Home exists to ship exactly three things:

  • Anthropic's skill-authoring skills — skill-creator, vendored via [[imports]], and its companion skill-reviewer.
  • A small set of ace-* helper skills that drive the harness itself.
  • Docs a new ACE user can read end-to-end in fifteen minutes.

It deliberately does not ship:

  • Language- or framework-specific coding conventions (Rust, Go, Python, frontend…).
  • Project-, team-, or org-specific rules (deploy targets, vendor names, thresholds).
  • Opinionated tool choices beyond the harness layer (CI vendor, package manager, formatter).

Those belong in your school. Keeping them out is what lets one baseline apply broadly.

Composition

Downstream schools inherit everything ACE Home ships with a wildcard import:

# downstream school.toml [[imports]] skill = "*" source = "ace-rs/school"

A project can exclude individual skills via ace.toml: exclude_skills, or shadow one by declaring a skill of the same name locally. Config layers user → project → school, so anything set per-project overrides the user default, which overrides the school's own school.toml.

The docs taxonomy this site is built from

The docs/ tree routes on one axis — the question an artifact answers — with placement forced by predicate-named folders and a first-match gate, not by after-the-fact judgment. Permanence isn't a second axis; it falls out of the folder.

FolderAnswersHandling
guides/how do I do X? (use or operate) living
vendor/what does their thing do? living; link-first
spec/how is ours built, and its surface? living
decisions/what did we rule, and why not? frozen; dated
scratch/what are we still working out? disposable; residual

scratch/ sits behind a toll — each file names why it isn't spec or a decision — so nothing lands there by default. CLAUDE.md/AGENTS.md is the agent index — no llms.txt, no human/ vs llm/ split.

This site is downstream

The docs/ tree is the source of truth; www/ (these pages) is a one-way, human-facing rewrite synthesized from it. Where docs/ sorts for maintainers, the site sorts for readers — collapsing several sources into one page, leading with concepts, dropping the disposable. Edit the docs, regenerate the page; never the reverse.
