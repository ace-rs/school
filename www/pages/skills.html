The skills

Eight skills, all loaded by default for any school that imports ACE Home. None are domain-specific — each covers a moment a raw coding session handles badly. Reach for them by the problem, not the mechanism.

SkillReach for it when…
ace-initfirst-time onboarding of a repo into ACE
acemoving the workflow forward a step
ace-savebefore a /clear, exit, or switch
ace-auditwork landed unreviewed, or you want a quality pass
ace-realigna rule keeps getting broken
ace-schoola fix should reach every project
ace-connecttwo local agents need to talk
ace-docsdurable artifacts are scattering

Onboarding & the workflow loop

ace-init is the cold-start: a repo just adopted ACE but nothing is tuned to it. It studies the repo once, narrows the active skills, refreshes the instructions file, and — on approval — seeds durable docs or a spec run. Run it on adoption, then hand off to the workflow.

ace advances the defined workflow — plan, write, test, audit — one step at a time. Invoke it again and again and the agent walks the stages itself, so you don't hand-direct each phase. At a session boundary it reads persisted state to resume mid-workflow instead of restarting.

Not losing work

ace-save is the deliberate checkpoint before you /clear, exit, or context-switch. Session memory and compaction are lossy and die with the session; ace-save writes to durable storage so the next session resumes from known-good state. (Notes only — not a substitute for git commit.)

ace-audit re-runs the audit against the relevant coding skills when a diff landed without passing through review, or when a large body of work just landed and you want a deliberate quality pass — catching structural problems while a rewrite is still cheap.

Making things stick & spread

ace-realign forces re-attention when the agent keeps violating a rule and restating it isn't working: it repeats the broken rule at the start or end of every message until you say stop.

ace-school proposes a fix back to the shared school via PR, so something you learned reaches every project that imports it instead of rotting in one repo or one machine's memory.

Plumbing

ace-connect gives two agents on the same machine (say Claude Code and Codex) a unix-socket bridge to message each other directly — fire-and-forget, single-user trust boundary. Not for intra-session, MCP, or cross-machine messaging.

ace-docs scaffolds a docs/ tree that nudges clean organization, and builds a downstream review site from it (this page is its output). See Architecture & design for the taxonomy.

skill-creator — Anthropic's authoritative skill-authoring skill — is also bundled, vendored via [[imports]] so the canonical workflow loads the moment anyone authors or revises a skill.
