ACE Home

The baseline school for ACE — the foundation every other school inherits from by default.

ACE is a thin coding harness on top of Claude Code, Codex, OpenCode, and friends. It wires shared skills, conventions, MCP servers, and session prompts into every project so your AI sessions all start from the same ground. A school is how that shared ground is packaged and distributed: a git repo a project subscribes to with ace setup <school>.

What a school can ship

  • Skills — progressively-disclosed instruction bundles the AI loads on trigger.
  • Session prompt — text prepended to every session.
  • MCP servers — remote endpoints wired into every subscriber.
  • Backend declarations — pre-rolled invocations of claude, codex, opencode, selectable with ace -b <name>.
  • Imports — other schools to inherit from; schools compose.
  • Conventions — house-style rules and durable docs read during work.

What ACE Home is for

ACE Home is the baseline: it makes a freshly-installed ACE useful out of the box, without forcing opinions that only fit one team or domain. It ships the small set of ace-* skills that drive the harness itself, the official Anthropic skill-creator, and documentation a new user can read end-to-end in fifteen minutes. Anything language-, team-, or project-specific belongs in your school, which imports this one and adds what your team actually needs. That layering is the point — ACE Home stays small and broadly applicable; downstream schools carry the opinions.

Quick start

In an existing project:

$ ace setup ace-rs/school $ ace

ace setup clones the school into a local cache, symlinks skills/ into the project, and writes ace.toml. ace launches the configured backend (Claude Code by default) with the school's skills, session prompt, and MCP servers wired in.
